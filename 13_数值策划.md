## 一、数值设计总则

### 1.1 核心原则

```
1. 策略 > 数值 > 氪金
   操作和策略始终是决定胜负的主因，数值提供上限空间，氪金缩短时间但不破坏平衡

2. 可感知的成长
   每次升级/突破必须让玩家在蹴鞠场上"感受到"变强（球更快、更弯、更猛）

3. 有意义的选择
   不存在"无脑堆某个属性就行"的最优解，不同Build需要不同属性倾向

4. N卡有用
   低稀有度角色通过独特机制在特定场景中不可替代

5. PVP公平优先
   天梯赛完全标准化，数值差异仅在江湖赛体现
```

### 1.2 数值缩写约定

| 缩写 | 含义 | 缩写 | 含义 |
|------|------|------|------|
| STR | 力量 | SPD | 速度 |
| TEC | 技巧 | DEF | 防御 |
| WIL | 意志 | GK | 守门 |
| Lv | 等级 | ★ | 星级 |
| E | 装备加成 | B | 羁绊加成 |

## 二、角色属性系统

### 2.1 基础属性公式

每个角色的最终属性由以下公式决定：

```
最终属性 = (基础属性 + 等级成长 + 装备加成) × 星级乘数 × 羁绊乘数

公式展开：
  Attr_final = (Attr_base + Lv × Growth_rate + E_flat) × Star_mult × Bond_mult
```

**参数定义：**

| 参数 | 说明 | 取值范围 |
|------|------|---------|
| Attr_base | 角色初始属性（Lv.1） | 由角色稀有度决定 |
| Growth_rate | 每级成长值 | SSR:0.8~1.2, SR:0.6~0.9, R:0.4~0.7, N:0.3~0.5 |
| E_flat | 装备固定加成 | 0 ~ 80（4件满级装备） |
| Star_mult | 星级倍率 | 见下表 |
| Bond_mult | 羁绊倍率 | 1.00 ~ 1.25 |

### 2.4 属性→物理引擎映射公式（完整版）

```python
def map_attr_to_physics(attr_final: dict) -> PhysicsParams:
    """从最终属性计算物理参数"""
    
    STR = attr_final['STR']
    SPD = attr_final['SPD']
    TEC = attr_final['TEC']
    DEF = attr_final['DEF']
    WIL = attr_final['WIL']
    GK  = attr_final['GK']
    
    # ===== 碰撞相关 =====
    mass = 1.5 + (STR / 200) * 3.0
    # Lv1 SSR范围: 1.5+(20~98)/200*3 = 1.8~2.97
    # Lv60★6 SSR范围: 1.5+(30~180)/200*3 = 1.95~4.2
    
    radius = 30 + (DEF / 200) * 25
    # Lv1 SSR范围: 30+(25~90)/200*25 = 33.1~41.3
    # Lv60★6 SSR范围: 30+(35~160)/200*25 = 34.4~50.0
    
    # ===== 运动相关 =====
    friction = 0.990 - (SPD / 200) * 0.04
    # 高SPD → 低摩擦 → 滑得更远
    # 范围: 0.970 ~ 0.990
    
    restitution = 0.70 + (SPD / 200) * 0.20
    # 高SPD → 高弹性 → 反弹更猛
    # 范围: 0.70 ~ 0.90
    
    max_speed = 1600 + (SPD / 200) * 400
    # 弹射最大速度限制
    # 范围: 1600 ~ 2000
    
    # ===== 技巧相关 =====
    spin_power = TEC / 200 * 2.0
    # 弧线球旋转强度, 范围: 0 ~ 2.0
    
    curve_control = TEC / 200
    # 弧线球可控度（越高弧线越稳定）
    # 范围: 0 ~ 1.0
    
    # ===== 战斗相关 =====
    kick_power = 1.0 + (STR / 200) * 0.5
    # 踢球力量加成, 范围: 1.0 ~ 1.5
    
    intercept_base = DEF / 200 * 0.50
    # 拦截基础概率, 范围: 0 ~ 0.50
    
    knockback_resist = DEF / 200 * 0.6
    # 被击退距离减免, 范围: 0 ~ 0.6
    
    # ===== 意志相关 =====
    crit_rate = WIL / 200 * 0.25
    # 暴击率, 范围: 0 ~ 0.25
    
    crit_damage = 1.5 + (WIL / 200) * 0.5
    # 暴击伤害倍率, 范围: 1.5 ~ 2.0
    
    skill_charge_rate = 1.0 + WIL / 200
    # 技能充能速度, 范围: 1.0 ~ 2.0
    
    # ===== 守门相关 =====
    save_radius = 60 + (GK / 100) * 40
    # 扑救范围, 范围: 60 ~ 100
    
    reaction_speed = 0.25 - (GK / 100) * 0.15
    # 反应延迟（秒）, 范围: 0.10 ~ 0.25
    
    save_accuracy = GK / 100
    # 扑救精准度（弧线球补偿系数）, 范围: 0 ~ 1.0
    
    return PhysicsParams(
        mass=mass, radius=radius,
        friction=friction, restitution=restitution, max_speed=max_speed,
        spin_power=spin_power, curve_control=curve_control,
        kick_power=kick_power, intercept_base=intercept_base,
        knockback_resist=knockback_resist,
        crit_rate=crit_rate, crit_damage=crit_damage,
        skill_charge_rate=skill_charge_rate,
        save_radius=save_radius, reaction_speed=reaction_speed,
        save_accuracy=save_accuracy
    )
```

> **为什么除以200而不是100？** 因为角色满级+装备+星级后属性可超过100（达到150~180），
> 用200作为分母保证物理参数不会溢出合理范围。

## 三、技能数值体系

### 3.1 技能等级成长

技能Lv.1→Lv.10的成长公式：

```
效果值 = 基础值 × (1 + (技能等级 - 1) × 0.10)
冷却回合 = max(基础冷却 - floor((技能等级 - 1) / 3), 最低冷却)
```

**示例：燕青「天巧花球」**

| 技能等级 | Spin倍率 | 马格努斯系数 | 冷却(回合) | 升级消耗 |
|---------|----------|------------|-----------|---------|
| Lv.1 | ×2.5 | 0.30 | 4 | — |
| Lv.2 | ×2.75 | 0.33 | 4 | 技能书×2 + 铜钱1000 |
| Lv.3 | ×3.0 | 0.36 | 4 | 技能书×3 + 铜钱2000 |
| Lv.4 | ×3.25 | 0.39 | 3 | 技能书×5 + 铜钱4000 |
| Lv.5 | ×3.5 | 0.42 | 3 | 技能书×8 + 铜钱7000 |
| Lv.6 | ×3.75 | 0.45 | 3 | 技能书×12 + 铜钱11000 |
| Lv.7 | ×4.0 | 0.48 | 2 | 技能书×18 + 铜钱16000 |
| Lv.8 | ×4.25 | 0.51 | 2 | 技能书×25 + 铜钱22000 |
| Lv.9 | ×4.5 | 0.54 | 2 | 技能书×35 + 铜钱30000 |
| Lv.10 | ×5.0 | 0.60 | 2 | 技能书×50 + 铜钱50000 |

**总消耗：158本技能书 + 143,000铜钱**

> PVP天梯赛锁定Lv.5，此等级为中间值——已有一定强度但不极端。

### 3.2 终极技能充能公式

```
充能回合 = 基础回合 × (1.0 / skill_charge_rate)
         = 基础回合 / (1.0 + WIL / 200)

例：燕青终极「蹴鞠真意」基础充能5回合
  WIL=78 → charge_rate = 1.39 → 实际充能 = 5 / 1.39 ≈ 3.6回合 → 取整4回合
  WIL=131(满级) → charge_rate = 1.66 → 实际充能 = 5 / 1.66 ≈ 3.0回合
```

### 3.3 暴击与伤害公式

```
射门力度 = ball.vel.magnitude × kick_power × combo_multiplier
暴击判定：random() < crit_rate → 力度 × crit_damage

combo_multiplier = 1.0 + combo_count × 0.10  （加法而非乘法，防爆炸）
  combo 0 = ×1.0
  combo 1 = ×1.1
  combo 2 = ×1.2
  combo 3 = ×1.3（2分进球阈值）
  combo 5 = ×1.5（3分进球阈值）

进球得分规则：
  combo < 3 → 1分
  3 ≤ combo < 5 → 2分
  combo ≥ 5 → 3分
```

### 3.4 拦截成功率公式

```
拦截率 = intercept_base - speed_penalty + terrain_bonus

  intercept_base = DEF / 200 × 0.50       （0 ~ 0.50）
  speed_penalty = ball_speed / 2000 × 0.30  （0 ~ 0.30）
  terrain_bonus = 是否在自家禁区 ? 0.10 : 0

理论范围：
  最低（低DEF + 高速球 + 非禁区）= 0.00 - 0.30 + 0 = 0%（不可为负，取0%）
  最高（高DEF + 低速球 + 禁区内）= 0.50 - 0.05 + 0.10 = 55%

设计目标：平均拦截率约25%~35%，保证进攻方有足够射门机会。
```

### 5.4 装备对战力的影响

```
一套满级红色装备提供的总属性：

这个占比是刻意控制的：

  太高 → 装备决定胜负，氪金感过重

### 6.2 保底机制数学分析

```
小保底：每10抽必出SR+（含SSR）
  → 如果10抽内没有SR+，第10抽强制变为SR
  → 实际SR+获取率 ≈ 12%（含保底提升）

大保底：累计80抽必出SSR
  → 累计概率：1-(0.98)^80 = 80.2%（不触发保底的概率为19.8%）
  → 有保底后100%在80抽内获得SSR

UP保底：歪了必捞
  → 50%概率出UP角色，50%出常驻SSR
  → 如果"歪了"（出非UP），下一个SSR 100%是UP
  → 最坏情况：160抽必得UP角色
  → 加权平均：约92抽获得UP角色

天井兑换：200抽可用"星宿令"兑换任意SSR
  → 每次抽卡获得1个星宿令（十连获得10个）
  → 200个星宿令 = 任选SSR（保底之保底）
```

### 11.2 赛季平衡更新节奏

```
