## 一、物理引擎总览

### 1.1 设计哲学

本游戏的物理系统不是模拟真实足球，而是 **"弹珠台 + 弹弓 + 足球"** 的混合体。
核心体验是：**拉弹射 → 武将冲出 → 撞球/撞人 → 连锁反应 → 射门得分**。

物理引擎的目标：
- **直觉性**：玩家拉弹弓就能理解操作
- **可预测性**：轨迹预测线让玩家能策略规划
- **爽快感**：连击/反弹/穿透产生 combo 的爽感
- **深度性**：角度、力度、反弹路线提供策略空间

### 1.2 坐标系与场地

## 二、核心物理对象

### 2.1 对象定义

游戏中所有物理对象都是 **圆形碰撞体（Circle Collider）**，简化计算并保证弹射手感。

```python
class PhysicsBody:
    """物理实体基类"""
    
    def __init__(self, x, y, radius, mass):
        self.pos = Vector2(x, y)          # 位置
        self.vel = Vector2(0, 0)          # 速度
        self.radius = radius              # 碰撞半径
        self.mass = mass                  # 质量（影响碰撞结果）
        self.friction = 0.98              # 每帧速度衰减系数
        self.restitution = 0.8            # 弹性系数（0=完全非弹性，1=完全弹性）
        self.is_active = False            # 是否处于运动状态
        self.angular_vel = 0.0            # 角速度（旋转/弧线球）
        self.spin = Vector2(0, 0)         # 自旋偏移量（马格努斯效应简化）


class HeroBody(PhysicsBody):
    """武将物理体"""
    
    def __init__(self, hero_data, x, y):
        super().__init__(x, y, radius=40, mass=hero_data.mass)
        self.hero_id = hero_data.id
        self.team = hero_data.team        # 0=我方, 1=敌方
        self.skill_modifier = None        # 技能对物理的修改器
        self.collision_count = 0          # 本回合碰撞次数
        self.is_launched = False          # 是否为本回合弹射的武将
        

class BallBody(PhysicsBody):
    """蹴鞠物理体"""
    
    def __init__(self, x, y):
        super().__init__(x, y, radius=20, mass=1.0)
        self.friction = 0.985            # 球的摩擦力比武将小（滚得更远）
        self.restitution = 0.9           # 球弹性更好
        self.last_touch_team = -1        # 最后触球队伍
        self.last_touch_hero = None      # 最后触球武将
        self.combo_count = 0             # 连击计数
        self.is_in_goal = False          # 是否进球


class WallBody:
    """场地边界（矩形）"""
    
    def __init__(self, x, y, width, height):
        self.pos = Vector2(x, y)
        self.width = width
        self.height = height
        self.restitution = 0.7           # 撞墙能量损失
```

## 三、弹射系统（核心交互）

### 3.1 输入阶段：拉弹弓

```
操作流程：
1. 玩家触摸武将 → 进入瞄准模式
2. 手指拖拽（远离武将方向）→ 显示弹弓拉伸动画
3. 实时显示预测轨迹（虚线 + 第一次反弹点）
4. 松手 → 武将沿反方向弹射出去
```

**力度计算：**

```python
class SlingShot:
    """弹弓系统"""
    
    MAX_PULL_DISTANCE = 200        # 最大拉伸距离（逻辑单位）
    MIN_PULL_DISTANCE = 20         # 最小有效拉伸
    MAX_LAUNCH_SPEED = 1800        # 最大弹射速度
    MIN_LAUNCH_SPEED = 200         # 最小弹射速度
    
    def calculate_launch(self, hero_pos, touch_start, touch_current):
        """
        计算弹射参数
        
        hero_pos:      武将位置
        touch_start:   触摸起始点（武将位置）
        touch_current: 当前手指位置
        """
        # 拉伸向量（从手指指向武将 = 弹射方向）
        pull_vec = hero_pos - touch_current
        pull_distance = pull_vec.length()
        
        # 限制最大拉伸
        pull_distance = clamp(pull_distance, 0, self.MAX_PULL_DISTANCE)
        
        if pull_distance < self.MIN_PULL_DISTANCE:
            return None  # 拉伸不够，取消弹射
        
        # 力度映射：使用二次曲线让手感更好（前段灵敏，后段平缓）
        power_ratio = (pull_distance / self.MAX_PULL_DISTANCE) ** 0.85
        launch_speed = lerp(self.MIN_LAUNCH_SPEED, self.MAX_LAUNCH_SPEED, power_ratio)
        
        # 弹射方向（单位向量）
        launch_dir = pull_vec.normalized()
        
        # 最终速度向量
        launch_velocity = launch_dir * launch_speed
        
        return LaunchData(
            velocity=launch_velocity,
            power_ratio=power_ratio,
            direction=launch_dir
        )
```

**力度曲线设计（关键手感参数）：**

```
速度
  ▲
  │            ╭───── MAX_LAUNCH_SPEED
  │          ╱
  │        ╱         ← power = distance^0.85
  │      ╱              前70%拉伸获得90%力量
  │    ╱                 减少"拉满才有用"的挫败感
  │  ╱
  │╱
  └──────────────────► 拉伸距离
  0                  MAX_PULL
```

> **为什么用 0.85 次方？**  
> 线性映射（1.0次方）会让前半段力度太弱，玩家需要精确拉到满才有效。
> 0.85次方让前70%拉伸就能获得约65%力量，降低操作门槛，但满拉仍有价值。
> 这个值需要在实际测试中微调（0.7~0.9 之间）。

### 3.2 轨迹预测线

```python
class TrajectoryPredictor:
    """轨迹预测显示"""
    
    PREDICT_STEPS = 120           # 预测帧数
    DOT_INTERVAL = 4              # 每N帧画一个点
    MAX_BOUNCES = 2               # 最多预测N次反弹
    
    def predict(self, hero, launch_velocity, ball, obstacles):
        """
        模拟弹射后的轨迹
        返回一系列点用于绘制虚线
        """
        points = []
        sim_pos = hero.pos.copy()
        sim_vel = launch_velocity.copy()
        bounces = 0
        
        for step in range(self.PREDICT_STEPS):
            # 简化的物理模拟（不计算完整碰撞）
            sim_pos += sim_vel * DT
            sim_vel *= hero.friction
            
            # 边界反弹
            if sim_pos.x - hero.radius < 0 or sim_pos.x + hero.radius > FIELD_WIDTH:
                sim_vel.x *= -1
                bounces += 1
            if sim_pos.y - hero.radius < -FIELD_HEIGHT/2 or sim_pos.y + hero.radius > FIELD_HEIGHT/2:
                sim_vel.y *= -1
                bounces += 1
            
            if bounces > self.MAX_BOUNCES:
                break
            
            if step % self.DOT_INTERVAL == 0:
                alpha = 1.0 - (step / self.PREDICT_STEPS) * 0.8  # 越远越淡
                points.append(TrajectoryPoint(sim_pos.copy(), alpha))
        
        return points
```

**视觉表现：**
- 预测线用 **水墨虚线** 风格
- 力度越大，虚线越长越亮
- 第一个碰撞目标（球/敌人）高亮显示
- 预测线不显示技能效果（保持策略悬念）

### 3.3 弹射后运动

```python
class PhysicsEngine:
    """物理引擎主循环"""
    
    DT = 1.0 / 60.0              # 固定时间步长 60fps
    VELOCITY_THRESHOLD = 5.0      # 低于此速度视为静止
    MAX_SIMULATION_TIME = 8.0     # 最长模拟时间（秒）防止无限弹
    
    def update(self, dt):
        """每帧物理更新"""
        
        for body in self.active_bodies:
            if not body.is_active:
                continue
            
            # 1. 施加速度
            body.pos += body.vel * dt
            
            # 2. 施加马格努斯效应（弧线球）
            if body.spin.length() > 0:
                body.vel += body.spin * dt * MAGNUS_COEFFICIENT
                body.spin *= 0.995  # 自旋衰减
            
            # 3. 施加摩擦力
            body.vel *= body.friction
            
            # 4. 检测是否静止
            if body.vel.length() < self.VELOCITY_THRESHOLD:
                body.vel = Vector2(0, 0)
                body.is_active = False
        
        # 5. 碰撞检测与响应
        self.resolve_collisions()
        
        # 6. 进球检测
        self.check_goal()
        
        # 7. 检查回合结束（所有物体静止）
        if self.all_stopped():
            self.end_turn()
```

## 四、碰撞系统

### 4.1 圆-圆碰撞检测

```python
def detect_circle_collision(a: PhysicsBody, b: PhysicsBody) -> bool:
    """检测两个圆形物体是否碰撞"""
    distance = (a.pos - b.pos).length()
    return distance < (a.radius + b.radius)


def resolve_circle_collision(a: PhysicsBody, b: PhysicsBody):
    """
    解析圆-圆碰撞
    使用动量守恒 + 弹性碰撞公式
    """
    # 碰撞法线（从a指向b）
    normal = (b.pos - a.pos).normalized()
    
    # 相对速度
    relative_vel = a.vel - b.vel
    
    # 法线方向的相对速度分量
    vel_along_normal = relative_vel.dot(normal)
    
    # 如果两物体正在分离，不处理
    if vel_along_normal < 0:
        return
    
    # 弹性系数取两者较小值
    restitution = min(a.restitution, b.restitution)
    
    # 冲量标量（基于动量守恒）
    # j = -(1 + e) * Vr·n / (1/ma + 1/mb)
    if b.mass == float('inf'):  # b是固定物体（墙/门柱）
        j = -(1 + restitution) * vel_along_normal / (1 / a.mass)
    elif a.mass == float('inf'):
        j = -(1 + restitution) * vel_along_normal / (1 / b.mass)
    else:
        j = -(1 + restitution) * vel_along_normal / (1/a.mass + 1/b.mass)
    
    # 施加冲量
    impulse = normal * j
    
    if a.mass != float('inf'):
        a.vel -= impulse / a.mass
    if b.mass != float('inf'):
        b.vel += impulse / b.mass
    
    # 位置修正（防止重叠/穿透）
    overlap = (a.radius + b.radius) - (b.pos - a.pos).length()
    if overlap > 0:
        correction = normal * (overlap / 2 + 1)  # +1 像素buffer
        if a.mass != float('inf'):
            a.pos -= correction
        if b.mass != float('inf'):
            b.pos += correction
```

### 4.2 圆-矩形碰撞（边界墙壁）

```python
def resolve_wall_collision(body: PhysicsBody):
    """处理与场地边界的碰撞"""
    
    # 左边界
    if body.pos.x - body.radius < 0:
        body.pos.x = body.radius
        body.vel.x = abs(body.vel.x) * body.restitution
    
    # 右边界
    if body.pos.x + body.radius > FIELD_WIDTH:
        body.pos.x = FIELD_WIDTH - body.radius
        body.vel.x = -abs(body.vel.x) * body.restitution
    
    # 上边界（对方底线，球门区域除外）
    if body.pos.y + body.radius > FIELD_HEIGHT / 2:
        if not is_in_goal_zone(body.pos, 'top'):
            body.pos.y = FIELD_HEIGHT / 2 - body.radius
            body.vel.y = -abs(body.vel.y) * body.restitution
    
    # 下边界（我方底线，球门区域除外）
    if body.pos.y - body.radius < -FIELD_HEIGHT / 2:
        if not is_in_goal_zone(body.pos, 'bottom'):
            body.pos.y = -FIELD_HEIGHT / 2 + body.radius
            body.vel.y = abs(body.vel.y) * body.restitution


def is_in_goal_zone(pos, side):
    """判断是否在球门区域内"""
    goal_left = (FIELD_WIDTH - GOAL_WIDTH) / 2
    goal_right = (FIELD_WIDTH + GOAL_WIDTH) / 2
    return goal_left < pos.x < goal_right
```

### 4.3 碰撞事件分类

碰撞不只是物理反弹，还会触发游戏逻辑：

```python
class CollisionEventType(Enum):
    HERO_HIT_BALL       = "hero_hit_ball"        # 武将踢球
    HERO_HIT_ALLY       = "hero_hit_ally"        # 撞到队友（触发传球配合）
    HERO_HIT_ENEMY      = "hero_hit_enemy"       # 撞到敌方（铲球/对抗）
    BALL_HIT_ALLY       = "ball_hit_ally"         # 球碰到己方武将（连击传球）
    BALL_HIT_ENEMY      = "ball_hit_enemy"        # 球碰到敌方武将（拦截判定）
    BALL_HIT_WALL       = "ball_hit_wall"         # 球撞墙反弹
    BALL_HIT_GOALPOST   = "ball_hit_goalpost"     # 球撞门柱
    BALL_ENTER_GOAL     = "ball_enter_goal"       # 进球！
    HERO_HIT_WALL       = "hero_hit_wall"         # 武将撞墙


def process_collision_event(event_type, body_a, body_b):
    """碰撞事件处理器"""
    
    match event_type:
        
        case CollisionEventType.HERO_HIT_BALL:
            # 武将踢球 — 球获得额外速度加成
            kick_power = body_a.hero_data.kick_power
            ball_speed_bonus = kick_power * 0.3
            body_b.vel *= (1 + ball_speed_bonus)
            
            # 记录触球信息
            body_b.last_touch_hero = body_a.hero_id
            body_b.last_touch_team = body_a.team
            
            # 检查是否触发武将主动技能
            if body_a.is_launched and body_a.skill_ready:
                trigger_active_skill(body_a, body_b)
            
            # 播放踢球特效和音效
            emit_signal("kick_ball", body_a, body_b)
        
        case CollisionEventType.BALL_HIT_ALLY:
            # 连击传球 — combo计数器+1
            body_b.combo_count += 1
            combo = body_b.combo_count
            
            # 连击加成：每次连击球速增加10%，伤害（射门力量）增加15%
            combo_speed_bonus = 1 + combo * 0.10
            combo_power_bonus = 1 + combo * 0.15
            body_b.vel *= combo_speed_bonus
            body_b.shot_power_multiplier = combo_power_bonus
            
            # 记录新的触球武将
            body_b.last_touch_hero = body_a.hero_id
            
            # 播放连击特效（连击数越高特效越华丽）
            emit_signal("combo_pass", body_a, body_b, combo)
        
        case CollisionEventType.HERO_HIT_ENEMY:
            # 对抗判定 — 比较双方物理属性
            attacker = body_a  # 运动中的武将
            defender = body_b  # 被撞的武将
            
            # 力量对比决定击退距离
            power_diff = attacker.hero_data.strength - defender.hero_data.strength
            knockback = max(0, power_diff * 0.5)
            
            collision_dir = (defender.pos - attacker.pos).normalized()
            defender.vel += collision_dir * knockback * 100
            defender.is_active = True
            
            # 眩晕判定：力量差 > 30 则眩晕1回合
            if power_diff > 30:
                defender.stun_turns = 1
                emit_signal("stun", defender)
            
            emit_signal("tackle", attacker, defender, power_diff)
        
        case CollisionEventType.BALL_HIT_ENEMY:
            # 拦截判定
            defender = body_b  # 敌方武将
            intercept_chance = defender.hero_data.defense / 100.0
            
            # 球速越高越难拦截
            ball_speed = body_a.vel.length()
            speed_penalty = min(0.5, ball_speed / self.MAX_LAUNCH_SPEED * 0.5)
            final_chance = intercept_chance - speed_penalty
            
            if random.random() < final_chance:
                # 拦截成功 — 球速大幅降低并偏转
                body_a.vel *= 0.2
                body_a.combo_count = 0  # 连击归零
                emit_signal("intercept_success", defender, body_a)
            else:
                # 拦截失败 — 球穿过，defender被击退
                knockback_dir = body_a.vel.normalized()
                defender.vel += knockback_dir * 200
                defender.is_active = True
                emit_signal("intercept_fail", defender, body_a)
        
        case CollisionEventType.BALL_ENTER_GOAL:
            # 进球！
            side = 'top' if body_a.pos.y > 0 else 'bottom'
            scoring_team = 0 if side == 'top' else 1  # 进上方球门=我方得分
            
            # 计算进球得分（基础1分 + combo加成）
            combo = body_a.combo_count
            goal_score = 1
            if combo >= 5:
                goal_score = 3  # 超级进球
            elif combo >= 3:
                goal_score = 2  # 精彩进球
            
            emit_signal("goal_scored", scoring_team, goal_score, 
                        body_a.last_touch_hero, combo)
```

## 五、高级物理机制

### 5.1 马格努斯效应（弧线球/旋转球）

某些技巧型武将可以给球施加旋转，产生弧线轨迹：

```python
MAGNUS_COEFFICIENT = 0.15  # 马格努斯效应强度

def apply_spin_to_ball(ball: BallBody, hero: HeroBody, kick_direction: Vector2):
    """
    给球施加旋转
    旋转方向垂直于踢球方向，强度由武将的 technique 属性决定
    """
    spin_strength = hero.hero_data.technique * 0.02
    
    # 旋转方向：踢球方向顺时针旋转90度
    spin_dir = Vector2(-kick_direction.y, kick_direction.x)
    
    ball.spin = spin_dir * spin_strength


# 在物理更新中应用马格努斯效应
def apply_magnus_effect(ball: BallBody, dt):
    """
    马格努斯效应：旋转的球会偏转
    F_magnus = spin × velocity × coefficient
    """
    if ball.spin.length() < 0.01:
        return
    
    # 偏转力垂直于当前运动方向
    magnus_force = Vector2(
        -ball.vel.y * ball.spin.x,
         ball.vel.x * ball.spin.y
    ) * MAGNUS_COEFFICIENT
    
    ball.vel += magnus_force * dt
    
    # 自旋随时间衰减
    ball.spin *= 0.995
```

**弧线球轨迹示意：**
```
直线球：  ● ─────────────────→ ⚽ 球门

弧线球：  ● ─────╲
                   ╲
                    ╲───→ ⚽ 球门
                    
         （绕过防守球员）
```

### 5.2 穿透机制

力量型武将的特殊能力 — 弹射不被阻挡：

```python
class PenetrationModifier:
    """穿透修改器"""
    
    def __init__(self, penetrate_count=1, damage_decay=0.2):
        self.penetrate_count = penetrate_count   # 可穿透次数
        self.damage_decay = damage_decay         # 每次穿透速度衰减比例
        self.current_penetrations = 0
    
    def on_collision(self, hero, other_body):
        """穿透碰撞时不反弹，直接穿过"""
        if self.current_penetrations < self.penetrate_count:
            self.current_penetrations += 1
            # 不改变武将速度方向，只衰减
            hero.vel *= (1 - self.damage_decay)
            # 被穿透的对象被击退
            knockback = hero.vel.normalized() * 300
            other_body.vel += knockback
            other_body.is_active = True
            return True  # 标记为穿透，跳过常规碰撞响应
        return False  # 穿透次数用完，正常碰撞
```

### 5.3 守门员AI物理

```python
class GoalkeeperAI:
    """
    守门员自动行为
    守门员不参与弹射，而是自动移动扑救
    """
    
    REACTION_SPEED = 400         # 移动速度
    DIVE_SPEED = 800             # 扑救速度
    SAVE_RADIUS = 80             # 扑救范围
    REACTION_DELAY = 0.15        # 反应延迟（秒）
    
    def __init__(self, hero_body, goal_y):
        self.body = hero_body
        self.goal_y = goal_y
        self.goal_center_x = FIELD_WIDTH / 2
        self.state = 'idle'          # idle / tracking / diving
        self.dive_target = None
        self.reaction_timer = 0
    
    def update(self, ball, dt):
        match self.state:
            
            case 'idle':
                # 缓慢回到球门中心
                target_x = self.goal_center_x
                self.body.pos.x = lerp(self.body.pos.x, target_x, 0.05)
                
                # 检测球是否朝球门运动
                if self.is_ball_threatening(ball):
                    self.state = 'tracking'
                    self.reaction_timer = self.REACTION_DELAY
            
            case 'tracking':
                self.reaction_timer -= dt
                if self.reaction_timer <= 0:
                    # 反应过来了，开始跟踪球的x位置
                    target_x = clamp(ball.pos.x, 
                                    self.goal_center_x - GOAL_WIDTH/2 + self.body.radius,
                                    self.goal_center_x + GOAL_WIDTH/2 - self.body.radius)
                    
                    move_dir = sign(target_x - self.body.pos.x)
                    self.body.pos.x += move_dir * self.REACTION_SPEED * dt
                
                # 球到达扑救距离时触发扑救
                dist_to_ball = (ball.pos - self.body.pos).length()
                if dist_to_ball < self.SAVE_RADIUS * 3:
                    self.state = 'diving'
                    self.dive_target = self.predict_ball_position(ball)
            
            case 'diving':
                # 向预判位置飞扑
                dive_dir = (self.dive_target - self.body.pos).normalized()
                self.body.pos += dive_dir * self.DIVE_SPEED * dt
                
                # 扑救判定
                dist_to_ball = (ball.pos - self.body.pos).length()
                if dist_to_ball < self.SAVE_RADIUS:
                    self.save_ball(ball)
                    self.state = 'idle'
    
    def is_ball_threatening(self, ball):
        """判断球是否正朝球门方向运动"""
        if self.goal_y > 0:  # 对方守门员（上方）
            return ball.vel.y > 100 and ball.pos.y > 0
        else:  # 我方守门员（下方）
            return ball.vel.y < -100 and ball.pos.y < 0
    
    def predict_ball_position(self, ball):
        """预判球到达球门线时的位置"""
        if abs(ball.vel.y) < 1:
            return ball.pos.copy()
        
        time_to_goal = abs((self.goal_y - ball.pos.y) / ball.vel.y)
        predicted_x = ball.pos.x + ball.vel.x * time_to_goal
        predicted_y = self.goal_y
        
        return Vector2(predicted_x, predicted_y)
    
    def save_ball(self, ball):
        """扑救成功"""
        # 守门员属性影响扑救后球的状态
        save_power = self.body.hero_data.goalkeeping
        
        # 扑出方向：朝场地中心偏转
        deflect_dir = Vector2(
            sign(FIELD_WIDTH/2 - ball.pos.x) * 0.7,
            -sign(self.goal_y) * 0.7
        ).normalized()
        
        ball.vel = deflect_dir * save_power * 3
        ball.combo_count = 0
        ball.spin = Vector2(0, 0)
        
        emit_signal("goalkeeper_save", self.body, ball)
```

### 5.4 技能对物理的修改器系统

技能不是独立的战斗系统，而是 **修改物理参数** ，让每次弹射都有不同物理表现：

```python
class PhysicsModifier:
    """物理修改器基类 — 技能通过修改物理参数来表现"""
    
    def on_launch(self, hero, velocity): 
        """弹射时修改初始速度"""
        return velocity
    
    def on_hero_collision(self, hero, other, normal):
        """武将碰撞时修改碰撞结果"""
        pass
    
    def on_ball_kick(self, hero, ball, kick_velocity):
        """踢球时修改球的速度"""
        return kick_velocity
    
    def on_ball_update(self, ball, dt):
        """每帧修改球的物理状态"""
        pass


class SkillPenetrate(PhysicsModifier):
    """林冲 — 豹子头冲射：穿透一切"""
    
    def on_launch(self, hero, velocity):
        hero.penetration = PenetrationModifier(penetrate_count=3, damage_decay=0.1)
        return velocity * 1.3  # 速度加成30%
    
    def on_ball_kick(self, hero, ball, kick_velocity):
        ball.penetration = PenetrationModifier(penetrate_count=2, damage_decay=0.15)
        return kick_velocity * 1.5


class SkillCurveBall(PhysicsModifier):
    """燕青 — 浪子花球：极致弧线"""
    
    def on_ball_kick(self, hero, ball, kick_velocity):
        spin_strength = hero.hero_data.technique * 0.05  # 比普通弧线强2.5倍
        kick_dir = kick_velocity.normalized()
        spin_dir = Vector2(-kick_dir.y, kick_dir.x)
        ball.spin = spin_dir * spin_strength
        return kick_velocity


class SkillBlackWhirlwind(PhysicsModifier):
    """李逵 — 黑旋风：AOE击飞"""
    
    def on_launch(self, hero, velocity):
        hero.aoe_radius = 120
        hero.aoe_knockback = 500
        return velocity * 0.9  # 速度略降但AOE补偿
    
    def on_hero_collision(self, hero, other, normal):
        # 撞到任何人都触发AOE
        for body in get_bodies_in_radius(hero.pos, hero.aoe_radius):
            if body != hero and body.team != hero.team:
                knockback_dir = (body.pos - hero.pos).normalized()
                body.vel += knockback_dir * hero.aoe_knockback
                body.is_active = True


class SkillDrunkFist(PhysicsModifier):
    """武松 — 醉拳盘带：球随机变向"""
    
    def on_ball_kick(self, hero, ball, kick_velocity):
        ball.drunk_changes = 2  # 还可以变向2次
        ball.drunk_angle_range = 45  # 最大变向角度（度）
        return kick_velocity
    
    def on_ball_update(self, ball, dt):
        if hasattr(ball, 'drunk_changes') and ball.drunk_changes > 0:
            # 每移动一定距离触发一次变向
            if ball.distance_since_last_change > 150:
                angle = random.uniform(-ball.drunk_angle_range, ball.drunk_angle_range)
                ball.vel = ball.vel.rotated(math.radians(angle))
                ball.drunk_changes -= 1
                ball.distance_since_last_change = 0
                emit_signal("drunk_direction_change", ball)


class SkillTimeStealer(PhysicsModifier):
    """时迁 — 鼓上蚤断球：偷球"""
    
    def on_ball_kick(self, hero, ball, kick_velocity):
        # 时迁不改变物理，而是设置一个flag
        # 球经过敌方时有概率"偷走"控球权
        ball.steal_chance = 0.5 + hero.hero_data.technique * 0.005
        return kick_velocity
```

## 六、回合流程与物理生命周期

```python
class TurnManager:
    """回合管理器"""
    
    def execute_turn(self):
        """一个完整回合的生命周期"""
        
        # ═══ Phase 1: 输入阶段 ═══
        # 玩家选择武将 → 拉弹弓 → 松手
        launch_data = yield WaitForPlayerInput()
        
        # ═══ Phase 2: 技能预处理 ═══
        hero = launch_data.hero
        velocity = launch_data.velocity
        
        if hero.active_skill and hero.skill_ready:
            modifier = hero.active_skill.get_modifier()
            velocity = modifier.on_launch(hero, velocity)
            hero.physics_modifier = modifier
        
        # ═══ Phase 3: 弹射 ═══
        hero.vel = velocity
        hero.is_active = True
        hero.is_launched = True
        
        # 弹射摄像机跟随
        camera.follow(hero, smooth=True)
        
        # ═══ Phase 4: 物理模拟循环 ═══
        simulation_time = 0
        while simulation_time < MAX_SIMULATION_TIME:
            physics_engine.update(DT)
            
            # 守门员AI更新
            for gk in goalkeepers:
                gk.update(ball, DT)
            
            # 检查进球
            if ball.is_in_goal:
                yield PlayGoalCelebration(ball)
                break
            
            # 检查所有物体是否静止
            if physics_engine.all_stopped():
                break
            
            simulation_time += DT
            yield WaitOneFrame()
        
        # ═══ Phase 5: 回合结算 ═══
        # 技能冷却更新
        for hero in current_team.heroes:
            hero.update_cooldowns()
        
        # combo统计
        max_combo = ball.combo_count
        if max_combo >= 3:
            yield ShowComboBonus(max_combo)
        
        # 重置回合状态
        ball.combo_count = 0
        hero.is_launched = False
        hero.collision_count = 0
        
        # 切换攻防（或继续进攻，取决于规则模式）
        self.switch_turn()
```

## 七、物理调参指南（给程序员的备注）

### 7.1 关键参数调整表

| 参数 | 影响 | 调大效果 | 调小效果 | 建议范围 |
|------|------|---------|---------|---------|
| `friction` | 运动持续时间 | 物体滑更远 | 物体很快停下 | 0.95~0.99 |
| `restitution` | 反弹力度 | 弹得更猛 | 反弹很弱 | 0.5~0.95 |
| `MAX_LAUNCH_SPEED` | 弹射力度上限 | 更暴力刺激 | 更精细策略 | 1200~2400 |
| `MAGNUS_COEFFICIENT` | 弧线球弯曲程度 | 球弯得离谱 | 几乎看不出弧线 | 0.08~0.25 |
| `VELOCITY_THRESHOLD` | 回合结束判定 | 快速结束 | 等很久才停 | 3~10 |
| `GK.REACTION_DELAY` | 守门员反应 | 更容易进球 | 守门员太强 | 0.1~0.3 |
| `combo_speed_bonus` | 连击速度加成 | Combo更爽但容易失控 | 连击感不明显 | 5%~15%/次 |
| `intercept_chance` | 拦截率 | 更难射门 | 防守形同虚设 | 20%~50%基础 |

### 7.2 手感黄金比例

```
理想的一次弹射应该持续 2~4 秒
理想的 combo 长度是 2~4 次连击
每场比赛理想时长 3~5 分钟
进球率应该在每 3~4 次弹射进 1 球
```

### 7.3 性能优化要点

```
1. 使用空间哈希网格（Spatial Hash Grid）加速碰撞检测
   - 将场地划分为 100x100 的网格
   - 只检测同一格子和相邻格子内的物体
   
2. 物理更新使用固定时间步长（Fixed Timestep）
   - 渲染帧率和物理帧率解耦
   - 物理固定 60fps，渲染尽可能高
   
3. 轨迹预测线使用简化物理
   - 不计算技能效果
   - 只计算边界反弹
   - 只预测弹射武将轨迹，不预测球
   
4. 碰撞检测使用 Broad Phase + Narrow Phase
   - Broad: AABB 快速排除
   - Narrow: 精确圆-圆检测
```

## 八、物理引擎技术选型建议

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **自研 2D 物理** | 完全可控，轻量 | 需要自己处理各种边界情况 | ⭐⭐⭐⭐ 推荐 |
| Unity 2D Physics (Box2D) | 成熟稳定，编辑器友好 | 对弹射游戏过于"真实"，需要大量调参 | ⭐⭐⭐ |
| Godot Physics 2D | 开源免费，够用 | 社区相对小 | ⭐⭐⭐ |
| SpriteKit (iOS原生) | 苹果原生性能最优 | 只能iOS，不跨平台 | ⭐⭐ |

> **推荐：** 使用 Unity/Godot 作为游戏引擎，但物理部分 **自己写**（如本文档所述），不用引擎自带物理。
> 原因：弹射类游戏需要的不是"真实物理"而是"好玩的物理"，自研更容易调出好手感。
