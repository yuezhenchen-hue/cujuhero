## 一、物理引擎总览

### 1.1 设计哲学

本游戏的物理系统不是模拟真实足球，而是 **"弹珠台 + 弹弓 + 足球"** 的混合体。
核心体验是：**拉弹射 → 武将冲出 → 撞球/撞人 → 连锁反应 → 射门得分**。

物理引擎的目标：
- **直觉性**：玩家拉弹弓就能理解操作
- **可预测性**：轨迹预测线让玩家能策略规划（PVP高段位逐步去除，提高操作天花板）
- **爽快感**：连击/反弹/穿透产生 combo 的爽感
- **深度性**：角度、力度、反弹路线提供策略空间
- **双向互动**：攻防双方在任何时刻都有操作可做，杜绝"看戏"体验

## 二、核心物理对象

### 2.1 对象定义

游戏中所有物理对象都是 **圆形碰撞体（Circle Collider）**，简化计算并保证弹射手感。

```python
class PhysicsBody:
    """物理实体基类"""
    
    def __init__(self, x, y, radius, mass):
        self.pos = Vector2(x, y)          # 位置
        self.vel = Vector2(0, 0)          # 速度
        self.radius = radius              # 碰撞半径
        self.mass = mass                  # 质量（影响碰撞结果）
        self.friction = 0.98              # 每帧速度衰减系数
        self.restitution = 0.8            # 弹性系数（0=完全非弹性，1=完全弹性）
        self.is_active = False            # 是否处于运动状态
        self.angular_vel = 0.0            # 角速度（旋转/弧线球）
        self.spin = Vector2(0, 0)         # 自旋偏移量（马格努斯效应简化）


class HeroBody(PhysicsBody):
    """武将物理体"""
    
    def __init__(self, hero_data, x, y):
        super().__init__(x, y, radius=40, mass=hero_data.mass)
        self.hero_id = hero_data.id
        self.team = hero_data.team        # 0=我方, 1=敌方
        self.skill_modifier = None        # 技能对物理的修改器
        self.collision_count = 0          # 本回合碰撞次数
        self.is_launched = False          # 是否为本回合弹射的武将
        

class BallBody(PhysicsBody):
    """蹴鞠物理体"""
    
    def __init__(self, x, y):
        super().__init__(x, y, radius=20, mass=1.0)
        self.friction = 0.985            # 球的摩擦力比武将小（滚得更远）
        self.restitution = 0.9           # 球弹性更好
        self.last_touch_team = -1        # 最后触球队伍
        self.last_touch_hero = None      # 最后触球武将
        self.combo_count = 0             # 连击计数
        self.is_in_goal = False          # 是否进球


class WallBody:
    """场地边界（矩形）"""
    
    def __init__(self, x, y, width, height):
        self.pos = Vector2(x, y)
        self.width = width
        self.height = height
        self.restitution = 0.7           # 撞墙能量损失
```

## 三、弹射系统（核心交互）

### 3.1 输入阶段：拉弹弓

```
操作流程：
1. 玩家触摸武将 → 进入瞄准模式
2. 手指拖拽（远离武将方向）→ 显示弹弓拉伸动画
3. 实时显示预测轨迹（虚线 + 第一次反弹点）
4. 松手 → 武将沿反方向弹射出去
```

**力度计算：**

```python
class SlingShot:
    """弹弓系统"""
    
    MAX_PULL_DISTANCE = 200        # 最大拉伸距离（逻辑单位）
    MIN_PULL_DISTANCE = 20         # 最小有效拉伸
    MAX_LAUNCH_SPEED = 1800        # 最大弹射速度
    MIN_LAUNCH_SPEED = 200         # 最小弹射速度
    
    def calculate_launch(self, hero_pos, touch_start, touch_current):
        """
        计算弹射参数
        
        hero_pos:      武将位置
        touch_start:   触摸起始点（武将位置）
        touch_current: 当前手指位置
        """
        # 拉伸向量（从手指指向武将 = 弹射方向）
        pull_vec = hero_pos - touch_current
        pull_distance = pull_vec.length()
        
        # 限制最大拉伸
        pull_distance = clamp(pull_distance, 0, self.MAX_PULL_DISTANCE)
        
        if pull_distance < self.MIN_PULL_DISTANCE:
            return None  # 拉伸不够，取消弹射
        
        # 力度映射：使用二次曲线让手感更好（前段灵敏，后段平缓）
        power_ratio = (pull_distance / self.MAX_PULL_DISTANCE) ** 0.85
        launch_speed = lerp(self.MIN_LAUNCH_SPEED, self.MAX_LAUNCH_SPEED, power_ratio)
        
        # 弹射方向（单位向量）
        launch_dir = pull_vec.normalized()
        
        # 最终速度向量
        launch_velocity = launch_dir * launch_speed
        
        return LaunchData(
            velocity=launch_velocity,
            power_ratio=power_ratio,
            direction=launch_dir
        )
```

**力度曲线设计（关键手感参数）：**

```
速度
  ▲
  │            ╭───── MAX_LAUNCH_SPEED
  │          ╱
  │        ╱         ← power = distance^0.85
  │      ╱              前70%拉伸获得90%力量
  │    ╱                 减少"拉满才有用"的挫败感
  │  ╱
  │╱
  └──────────────────► 拉伸距离
  0                  MAX_PULL
```

> **为什么用 0.85 次方？**  
> 线性映射（1.0次方）会让前半段力度太弱，玩家需要精确拉到满才有效。
> 0.85次方让前70%拉伸就能获得约65%力量，降低操作门槛，但满拉仍有价值。
> 这个值需要在实际测试中微调（0.7~0.9 之间）。

### 3.2 轨迹预测线（分级系统）

为了同时满足新手友好和竞技深度，预测线根据**游戏模式和段位**动态调整：

```python
class PredictionLevel(Enum):
    FULL = "full"             # 完整预测线（含2次反弹）
    MEDIUM = "medium"         # 只显示50%长度，1次反弹
    DIRECTION_ONLY = "dir"    # 只显示方向箭头，无距离无反弹
    NONE = "none"             # 无预测线（纯硬核）


def get_prediction_level(game_mode, pvp_rank=None, rogue_floor=None):
    """根据模式和段位返回预测线等级"""
    if game_mode == "pve_normal":
        return PredictionLevel.FULL
    elif game_mode == "pve_hard":
        return PredictionLevel.MEDIUM
    elif game_mode == "roguelike":
        if rogue_floor <= 2:
            return PredictionLevel.FULL
        elif rogue_floor == 3:
            return PredictionLevel.MEDIUM
        else:
            return PredictionLevel.NONE
    elif game_mode == "pvp":
        if pvp_rank <= RankTier.GOLD:
            return PredictionLevel.FULL
        elif pvp_rank <= RankTier.DIAMOND:
            return PredictionLevel.MEDIUM
        elif pvp_rank <= RankTier.MASTER:
            return PredictionLevel.DIRECTION_ONLY
        else:
            return PredictionLevel.NONE


class TrajectoryPredictor:
    """轨迹预测显示——支持分级精度"""
    
    FULL_STEPS = 120
    MEDIUM_STEPS = 60
    DOT_INTERVAL = 4
    
    LEVEL_CONFIG = {
        PredictionLevel.FULL:    {"steps": 120, "bounces": 2, "show_target": True},
        PredictionLevel.MEDIUM:  {"steps": 60,  "bounces": 1, "show_target": False},
        PredictionLevel.DIRECTION_ONLY: {"steps": 0, "bounces": 0, "show_target": False},
        PredictionLevel.NONE:    {"steps": 0, "bounces": 0, "show_target": False},
    }
    
    def predict(self, hero, launch_velocity, ball, obstacles, level: PredictionLevel):
        if level == PredictionLevel.NONE:
            return []
        
        if level == PredictionLevel.DIRECTION_ONLY:
            arrow_end = hero.pos + launch_velocity.normalized() * 80
            return [TrajectoryPoint(hero.pos.copy(), 1.0),
                    TrajectoryPoint(arrow_end, 0.6)]
        
        config = self.LEVEL_CONFIG[level]
        points = []
        sim_pos = hero.pos.copy()
        sim_vel = launch_velocity.copy()
        bounces = 0
        
        for step in range(config["steps"]):
            sim_pos += sim_vel * DT
            sim_vel *= hero.friction
            
            if sim_pos.x - hero.radius < 0 or sim_pos.x + hero.radius > FIELD_WIDTH:
                sim_vel.x *= -1
                bounces += 1
            if sim_pos.y - hero.radius < -FIELD_HEIGHT/2 or sim_pos.y + hero.radius > FIELD_HEIGHT/2:
                sim_vel.y *= -1
                bounces += 1
            
            if bounces > config["bounces"]:
                break
            
            if step % self.DOT_INTERVAL == 0:
                alpha = 1.0 - (step / config["steps"]) * 0.8
                points.append(TrajectoryPoint(sim_pos.copy(), alpha))
        
        return points
```

> **竞技意义**：从"有线"到"无线"的适应过程本身是成长快感，高段位对局观赏性极强——"他怎么算出这个反弹角度的？！"

- 预测线用 **水墨虚线** 风格
- 力度越大，虚线越长越亮
- FULL级别：第一个碰撞目标（球/敌人）高亮显示
- 预测线不显示技能效果（保持策略悬念）
- DIRECTION_ONLY级别：仅显示短箭头，带水墨笔触动效

### 3.3 弹射后运动

```python
class PhysicsEngine:
    """物理引擎主循环"""
    
    DT = 1.0 / 60.0              # 固定时间步长 60fps
    VELOCITY_THRESHOLD = 5.0      # 低于此速度视为静止
    MAX_SIMULATION_TIME = 8.0     # 最长模拟时间（秒）防止无限弹
    
    def update(self, dt):
        """每帧物理更新"""
        
        for body in self.active_bodies:
            if not body.is_active:
                continue
            
            # 1. 施加速度
            body.pos += body.vel * dt
            
            # 2. 施加马格努斯效应（弧线球）
            if body.spin.length() > 0:
                body.vel += body.spin * dt * MAGNUS_COEFFICIENT
                body.spin *= 0.995  # 自旋衰减
            
            # 3. 施加摩擦力
            body.vel *= body.friction
            
            # 4. 检测是否静止
            if body.vel.length() < self.VELOCITY_THRESHOLD:
                body.vel = Vector2(0, 0)
                body.is_active = False
        
        # 5. 碰撞检测与响应
        self.resolve_collisions()
        
        # 6. 进球检测
        self.check_goal()
        
        # 7. 检查回合结束（所有物体静止）
        if self.all_stopped():
            self.end_turn()
```

## 四、碰撞系统

### 4.1 圆-圆碰撞检测

```python
def detect_circle_collision(a: PhysicsBody, b: PhysicsBody) -> bool:
    """检测两个圆形物体是否碰撞"""
    distance = (a.pos - b.pos).length()
    return distance < (a.radius + b.radius)


def resolve_circle_collision(a: PhysicsBody, b: PhysicsBody):
    """
    解析圆-圆碰撞
    使用动量守恒 + 弹性碰撞公式
    """
    # 碰撞法线（从a指向b）
    normal = (b.pos - a.pos).normalized()
    
    # 相对速度
    relative_vel = a.vel - b.vel
    
    # 法线方向的相对速度分量
    vel_along_normal = relative_vel.dot(normal)
    
    # 如果两物体正在分离，不处理
    if vel_along_normal < 0:
        return
    
    # 弹性系数取两者较小值
    restitution = min(a.restitution, b.restitution)
    
    # 冲量标量（基于动量守恒）
    # j = -(1 + e) * Vr·n / (1/ma + 1/mb)
    if b.mass == float('inf'):  # b是固定物体（墙/门柱）
        j = -(1 + restitution) * vel_along_normal / (1 / a.mass)
    elif a.mass == float('inf'):
        j = -(1 + restitution) * vel_along_normal / (1 / b.mass)
    else:
        j = -(1 + restitution) * vel_along_normal / (1/a.mass + 1/b.mass)
    
    # 施加冲量
    impulse = normal * j
    
    if a.mass != float('inf'):
        a.vel -= impulse / a.mass
    if b.mass != float('inf'):
        b.vel += impulse / b.mass
    
    # 位置修正（防止重叠/穿透）
    overlap = (a.radius + b.radius) - (b.pos - a.pos).length()
    if overlap > 0:
        correction = normal * (overlap / 2 + 1)  # +1 像素buffer
        if a.mass != float('inf'):
            a.pos -= correction
        if b.mass != float('inf'):
            b.pos += correction
```

### 4.2 圆-矩形碰撞（边界墙壁）

```python
def resolve_wall_collision(body: PhysicsBody):
    """处理与场地边界的碰撞"""
    
    # 左边界
    if body.pos.x - body.radius < 0:
        body.pos.x = body.radius
        body.vel.x = abs(body.vel.x) * body.restitution
    
    # 右边界
    if body.pos.x + body.radius > FIELD_WIDTH:
        body.pos.x = FIELD_WIDTH - body.radius
        body.vel.x = -abs(body.vel.x) * body.restitution
    
    # 上边界（对方底线，球门区域除外）
    if body.pos.y + body.radius > FIELD_HEIGHT / 2:
        if not is_in_goal_zone(body.pos, 'top'):
            body.pos.y = FIELD_HEIGHT / 2 - body.radius
            body.vel.y = -abs(body.vel.y) * body.restitution
    
    # 下边界（我方底线，球门区域除外）
    if body.pos.y - body.radius < -FIELD_HEIGHT / 2:
        if not is_in_goal_zone(body.pos, 'bottom'):
            body.pos.y = -FIELD_HEIGHT / 2 + body.radius
            body.vel.y = abs(body.vel.y) * body.restitution


def is_in_goal_zone(pos, side):
    """判断是否在球门区域内"""
    goal_left = (FIELD_WIDTH - GOAL_WIDTH) / 2
    goal_right = (FIELD_WIDTH + GOAL_WIDTH) / 2
    return goal_left < pos.x < goal_right
```

### 4.3 碰撞事件分类

碰撞不只是物理反弹，还会触发游戏逻辑：

```python
class CollisionEventType(Enum):
    HERO_HIT_BALL       = "hero_hit_ball"        # 武将踢球
    HERO_HIT_ALLY       = "hero_hit_ally"        # 撞到队友（触发传球配合）
    HERO_HIT_ENEMY      = "hero_hit_enemy"       # 撞到敌方（铲球/对抗）
    BALL_HIT_ALLY       = "ball_hit_ally"         # 球碰到己方武将（连击传球）
    BALL_HIT_ENEMY      = "ball_hit_enemy"        # 球碰到敌方武将（拦截判定）
    BALL_HIT_WALL       = "ball_hit_wall"         # 球撞墙反弹
    BALL_HIT_GOALPOST   = "ball_hit_goalpost"     # 球撞门柱
    BALL_ENTER_GOAL     = "ball_enter_goal"       # 进球！
    HERO_HIT_WALL       = "hero_hit_wall"         # 武将撞墙


def process_collision_event(event_type, body_a, body_b):
    """碰撞事件处理器"""
    
    match event_type:
        
        case CollisionEventType.HERO_HIT_BALL:
            # 武将踢球 — 球获得额外速度加成
            kick_power = body_a.hero_data.kick_power
            ball_speed_bonus = kick_power * 0.3
            body_b.vel *= (1 + ball_speed_bonus)
            
            # 记录触球信息
            body_b.last_touch_hero = body_a.hero_id
            body_b.last_touch_team = body_a.team
            
            # 检查是否触发武将主动技能
            if body_a.is_launched and body_a.skill_ready:
                trigger_active_skill(body_a, body_b)
            
            # 播放踢球特效和音效
            emit_signal("kick_ball", body_a, body_b)
        
        case CollisionEventType.BALL_HIT_ALLY:
            # 连击传球 — combo计数器+1
            body_b.combo_count += 1
            combo = body_b.combo_count
            
            # 连击加成：每次连击球速增加10%，伤害（射门力量）增加15%
            combo_speed_bonus = 1 + combo * 0.10
            combo_power_bonus = 1 + combo * 0.15
            body_b.vel *= combo_speed_bonus
            body_b.shot_power_multiplier = combo_power_bonus
            
            # 记录新的触球武将
            body_b.last_touch_hero = body_a.hero_id
            
            # 播放连击特效（连击数越高特效越华丽）
            emit_signal("combo_pass", body_a, body_b, combo)
        
        case CollisionEventType.HERO_HIT_ENEMY:
            # 对抗判定 — 比较双方物理属性
            attacker = body_a  # 运动中的武将
            defender = body_b  # 被撞的武将
            
            # 力量对比决定击退距离
            power_diff = attacker.hero_data.strength - defender.hero_data.strength
            knockback = max(0, power_diff * 0.5)
            
            collision_dir = (defender.pos - attacker.pos).normalized()
            defender.vel += collision_dir * knockback * 100
            defender.is_active = True
            
            # 眩晕判定：力量差 > 30 则眩晕1回合
            if power_diff > 30:
                defender.stun_turns = 1
                emit_signal("stun", defender)
            
            emit_signal("tackle", attacker, defender, power_diff)
        
        case CollisionEventType.BALL_HIT_ENEMY:
            # 拦截判定
            defender = body_b  # 敌方武将
            intercept_chance = defender.hero_data.defense / 100.0
            
            # 球速越高越难拦截
            ball_speed = body_a.vel.length()
            speed_penalty = min(0.5, ball_speed / self.MAX_LAUNCH_SPEED * 0.5)
            final_chance = intercept_chance - speed_penalty
            
            if random.random() < final_chance:
                # 拦截成功 — 球速大幅降低并偏转
                body_a.vel *= 0.2
                body_a.combo_count = 0  # 连击归零
                emit_signal("intercept_success", defender, body_a)
            else:
                # 拦截失败 — 球穿过，defender被击退
                knockback_dir = body_a.vel.normalized()
                defender.vel += knockback_dir * 200
                defender.is_active = True
                emit_signal("intercept_fail", defender, body_a)
        
        case CollisionEventType.BALL_ENTER_GOAL:
            # 进球！
            side = 'top' if body_a.pos.y > 0 else 'bottom'
            scoring_team = 0 if side == 'top' else 1  # 进上方球门=我方得分
            
            # 计算进球得分（基础1分 + combo加成）
            combo = body_a.combo_count
            goal_score = 1
            if combo >= 5:
                goal_score = 3  # 超级进球
            elif combo >= 3:
                goal_score = 2  # 精彩进球
            
            emit_signal("goal_scored", scoring_team, goal_score, 
                        body_a.last_touch_hero, combo)
```

### 5.1 马格努斯效应（弧线球/旋转球）

```python
MAGNUS_COEFFICIENT = 0.15  # 马格努斯效应强度

def apply_spin_to_ball(ball: BallBody, hero: HeroBody, kick_direction: Vector2):
    """
    给球施加旋转
    旋转方向垂直于踢球方向，强度由武将的 technique 属性决定
    """
    spin_strength = hero.hero_data.technique * 0.02
    
    # 旋转方向：踢球方向顺时针旋转90度
    spin_dir = Vector2(-kick_direction.y, kick_direction.x)
    
    ball.spin = spin_dir * spin_strength


# 在物理更新中应用马格努斯效应
def apply_magnus_effect(ball: BallBody, dt):
    """
    马格努斯效应：旋转的球会偏转
    F_magnus = spin × velocity × coefficient
    """
    if ball.spin.length() < 0.01:
        return
    
    # 偏转力垂直于当前运动方向
    magnus_force = Vector2(
        -ball.vel.y * ball.spin.x,
         ball.vel.x * ball.spin.y
    ) * MAGNUS_COEFFICIENT
    
    ball.vel += magnus_force * dt
    
    # 自旋随时间衰减
    ball.spin *= 0.995
```

```python
class PenetrationModifier:
    """穿透修改器"""
    
    def __init__(self, penetrate_count=1, damage_decay=0.2):
        self.penetrate_count = penetrate_count   # 可穿透次数
        self.damage_decay = damage_decay         # 每次穿透速度衰减比例
        self.current_penetrations = 0
    
    def on_collision(self, hero, other_body):
        """穿透碰撞时不反弹，直接穿过"""
        if self.current_penetrations < self.penetrate_count:
            self.current_penetrations += 1
            # 不改变武将速度方向，只衰减
            hero.vel *= (1 - self.damage_decay)
            # 被穿透的对象被击退
            knockback = hero.vel.normalized() * 300
            other_body.vel += knockback
            other_body.is_active = True
            return True  # 标记为穿透，跳过常规碰撞响应
        return False  # 穿透次数用完，正常碰撞
```

### 5.3 守门员系统（AI + 手动扑救）

守门员有两种模式：**AI自动扑救**（PVE / PVP低段位）和 **手动扑救**（PVP钻石以上）。

#### 5.3.1 守门员AI（改进版：弧线球预判）

```python
class GoalkeeperAI:
    """
    守门员自动行为
    v1.1改进：predict_ball_position 考虑马格努斯效应（弧线球偏移）
    """
    
    REACTION_SPEED = 400
    DIVE_SPEED = 800
    SAVE_RADIUS = 80
    REACTION_DELAY = 0.15
    
    def __init__(self, hero_body, goal_y):
        self.body = hero_body
        self.goal_y = goal_y
        self.goal_center_x = FIELD_WIDTH / 2
        self.state = 'idle'
        self.dive_target = None
        self.reaction_timer = 0
    
    def update(self, ball, dt):
        match self.state:
            case 'idle':
                target_x = self.goal_center_x
                self.body.pos.x = lerp(self.body.pos.x, target_x, 0.05)
                if self.is_ball_threatening(ball):
                    self.state = 'tracking'
                    self.reaction_timer = self.REACTION_DELAY
            
            case 'tracking':
                self.reaction_timer -= dt
                if self.reaction_timer <= 0:
                    target_x = clamp(ball.pos.x,
                                    self.goal_center_x - GOAL_WIDTH/2 + self.body.radius,
                                    self.goal_center_x + GOAL_WIDTH/2 - self.body.radius)
                    move_dir = sign(target_x - self.body.pos.x)
                    self.body.pos.x += move_dir * self.REACTION_SPEED * dt
                
                dist_to_ball = (ball.pos - self.body.pos).length()
                if dist_to_ball < self.SAVE_RADIUS * 3:
                    self.state = 'diving'
                    self.dive_target = self.predict_ball_position(ball)
            
            case 'diving':
                dive_dir = (self.dive_target - self.body.pos).normalized()
                self.body.pos += dive_dir * self.DIVE_SPEED * dt
                
                dist_to_ball = (ball.pos - self.body.pos).length()
                if dist_to_ball < self.SAVE_RADIUS:
                    self.save_ball(ball)
                    self.state = 'idle'
    
    def is_ball_threatening(self, ball):
        if self.goal_y > 0:
            return ball.vel.y > 100 and ball.pos.y > 0
        else:
            return ball.vel.y < -100 and ball.pos.y < 0
    
    def predict_ball_position(self, ball):
        """
        v1.1改进：考虑马格努斯效应的弧线球预判
        GK属性越高，预判偏移补偿越精准
        """
        if abs(ball.vel.y) < 1:
            return ball.pos.copy()
        
        time_to_goal = abs((self.goal_y - ball.pos.y) / ball.vel.y)
        
        # 基础线性预判
        predicted_x = ball.pos.x + ball.vel.x * time_to_goal
        
        # 弧线球补偿：根据球的自旋估算偏移量
        if ball.spin.length() > 0.01:
            magnus_offset_x = (-ball.vel.y * ball.spin.x) * MAGNUS_COEFFICIENT * (time_to_goal ** 2) * 0.5
            
            # GK属性决定预判精度（0~100映射到0~1）
            gk_accuracy = self.body.hero_data.goalkeeping / 100.0
            predicted_x += magnus_offset_x * gk_accuracy
        
        predicted_y = self.goal_y
        return Vector2(
            clamp(predicted_x,
                  self.goal_center_x - GOAL_WIDTH/2,
                  self.goal_center_x + GOAL_WIDTH/2),
            predicted_y
        )
    
    def save_ball(self, ball):
        save_power = self.body.hero_data.goalkeeping
        deflect_dir = Vector2(
            sign(FIELD_WIDTH/2 - ball.pos.x) * 0.7,
            -sign(self.goal_y) * 0.7
        ).normalized()
        ball.vel = deflect_dir * save_power * 3
        ball.combo_count = 0
        ball.spin = Vector2(0, 0)
        emit_signal("goalkeeper_save", self.body, ball)
```

```python
class ManualGoalkeeperSave:
    """
    手动扑救 — 对手射门时替代AI的玩家操作
    设计理念：让丢球/扑救成为玩家的决策，而不是AI掷骰子
    """
    
    SWIPE_WINDOW = 0.8           # 操作窗口（秒）
    DIVE_LEFT = -1
    DIVE_RIGHT = 1
    DIVE_STAY = 0
    
    SAVE_ZONES = {
        "left":   (-GOAL_WIDTH/2, -GOAL_WIDTH/6),
        "center": (-GOAL_WIDTH/6,  GOAL_WIDTH/6),
        "right":  ( GOAL_WIDTH/6,  GOAL_WIDTH/2),
    }
    
    def __init__(self, gk_body, goal_y):
        self.gk = gk_body
        self.goal_y = goal_y
        self.active = False
        self.player_input = None
        self.timer = 0
    
    def activate(self, ball):
        """球朝球门飞来时激活手动扑救"""
        self.active = True
        self.timer = self.SWIPE_WINDOW
        self.player_input = None
        emit_signal("manual_save_prompt", self.gk)
    
    def receive_input(self, swipe_direction):
        """接收玩家滑动输入: -1=左, 0=不动, 1=右"""
        self.player_input = swipe_direction
    
    def resolve(self, ball):
        """
        判定扑救结果
        猜对方向 → 扑救率大幅提高（受GK属性加成）
        猜错方向 → 几乎必进
        """
        ball_zone = self._get_ball_zone(ball)
        dive_zone = self._input_to_zone(self.player_input)
        
        gk_stat = self.gk.hero_data.goalkeeping
        
        if ball_zone == dive_zone:
            save_chance = 0.60 + gk_stat / 100.0 * 0.35   # 60%~95%
        elif abs(ball_zone - dive_zone) == 1:
            save_chance = 0.15 + gk_stat / 100.0 * 0.15   # 15%~30%
        else:
            save_chance = 0.02                              # 完全猜错，几乎必进
        
        if random.random() < save_chance:
            self._execute_save(ball)
            return True
        return False
    
    def _get_ball_zone(self, ball):
        relative_x = ball.pos.x - FIELD_WIDTH / 2
        if relative_x < -GOAL_WIDTH / 6:
            return -1
        elif relative_x > GOAL_WIDTH / 6:
            return 1
        return 0
    
    def _input_to_zone(self, inp):
        if inp is None:
            return 0
        return inp
    
    def _execute_save(self, ball):
        ball.vel *= 0.15
        ball.combo_count = 0
        ball.spin = Vector2(0, 0)
        emit_signal("manual_save_success", self.gk, ball)
```

### 5.4 防守回应系统（v1.1新增）

**核心理念：对手弹射时防守方也有操作可做，消灭"50%时间看戏"。**

```python
class DefenseResponseSystem:
    """
    防守回应 — 对手弹射进入物理模拟后，防守方可执行的操作
    每场比赛限制次数，避免过度干预物理模拟的随机性
    """
    
    MAX_REPOSITIONS_PER_MATCH = 4     # 每场最多4次紧急变阵
    REPOSITION_WINDOW = 1.0           # 操作窗口（对手弹射后1秒内）
    MAX_MOVE_DISTANCE = 80            # 后卫最大移动距离
    
    SKILL_ACTIVATION_WINDOW = 0.5     # 触发式被动技能的操作窗口
    
    def __init__(self, team):
        self.team = team
        self.repositions_remaining = self.MAX_REPOSITIONS_PER_MATCH
        self.window_timer = 0
        self.can_respond = False
    
    def on_opponent_launch(self):
        """对手弹射瞬间触发防守窗口"""
        if self.repositions_remaining > 0:
            self.can_respond = True
            self.window_timer = self.REPOSITION_WINDOW
            emit_signal("defense_window_open", self.team)
    
    def update(self, dt):
        if self.can_respond:
            self.window_timer -= dt
            if self.window_timer <= 0:
                self.can_respond = False
                emit_signal("defense_window_closed", self.team)
    
    def execute_reposition(self, defender, target_pos):
        """
        紧急变阵：拖拽一名后卫短距离移动
        - 只有DF位置的武将可被拖拽
        - 移动距离不超过MAX_MOVE_DISTANCE
        - 操作窗口极短（1秒），考验反应力
        """
        if not self.can_respond:
            return False
        if defender.position != 'DF':
            return False
        
        move_vec = target_pos - defender.pos
        if move_vec.length() > self.MAX_MOVE_DISTANCE:
            move_vec = move_vec.normalized() * self.MAX_MOVE_DISTANCE
        
        defender.pos += move_vec
        self.repositions_remaining -= 1
        self.can_respond = False
        
        emit_signal("defender_repositioned", defender, move_vec)
        return True
    
    def activate_defender_skill(self, defender):
        """
        触发式被动技能 — 防守方主动点击后卫激活
        例如鲁智深的禅域减速不再自动生效，而是玩家在
        对手射门时点击激活，增加时机判断的操作深度
        """
        if not defender.passive_skill or not defender.passive_skill.is_triggered:
            return False
        if defender.passive_skill.is_on_cooldown():
            return False
        
        defender.passive_skill.activate()
        emit_signal("defender_skill_activated", defender)
        return True
```

**防守回应交互总结：**

```
对手弹射 → 物理模拟开始
  │
  ├─ 第0~1秒：【紧急变阵窗口】
  │   → 拖拽一名后卫短距离移动（每场限4次）
  │   → 或点击后卫激活触发式被动技能
  │
  ├─ 球飞向我方半场：【手动扑救窗口】（仅PVP钻石以上）
  │   → 向左/右滑动控制守门员扑救方向
  │
  └─ 物理模拟结束 → 回合结算
```

```python
    惩罚消极比赛，鼓励积极进攻
    """

    }

        self.turns_since_shot = {0: 0, 1: 0}   # 每队的无射门计数

            self.turns_since_shot[team] += 1

            self.turns_since_shot[team] = 0

            self.turns_ball_untouched = 0
        else:
            self.turns_ball_untouched += 1
        
        if self.turns_ball_untouched >= self.IDLE_BALL_TURNS:
            self._advance_ball()
            self.turns_ball_untouched = 0
    
    def get_comeback_buffs(self, team, score_diff):

        buffs = {}
        for threshold, bonuses in self.COMEBACK_THRESHOLDS.items():
            if score_diff >= threshold:
                buffs.update(bonuses)
        return buffs
    
    def _award_free_kick(self, team):
```

```python
    """回合管理器 — v1.1支持半场制、防守回应、手动扑救"""
    
    HALF_TURNS = 5           # 每半场5回合
    HALFTIME_DURATION = 30   # 中场休息30秒
    
    def __init__(self, team_a, team_b, game_mode, pvp_rank=None):
        self.teams = [team_a, team_b]
        self.game_mode = game_mode
        self.pvp_rank = pvp_rank
        self.half = 1                 # 1=上半场, 2=下半场
        self.turn_in_half = 0
        self.first_team_this_half = 0 # 上半场A先手
        
        self.defense_response = DefenseResponseSystem(None)
        self.anti_passive = AntiPassiveSystem()
        self.prediction_level = get_prediction_level(game_mode, pvp_rank)
        
        # 手动扑救判定
        self.manual_save_enabled = (
            game_mode == "pvp" and pvp_rank and pvp_rank >= RankTier.DIAMOND
        )
    
    def run_match(self):
        """完整比赛流程"""
        
        # ═══ 上半场 ═══
        self.half = 1
        self.first_team_this_half = 0   # A先手
        yield from self._run_half()
        
        # ═══ 中场休息 ═══
        yield HalftimeBreak(self.HALFTIME_DURATION)
        
        # ═══ 下半场 ═══
        self.half = 2
        self.first_team_this_half = 1   # B先手（先手互换）
        self.defense_response.repositions_remaining = 4  # 刷新防守次数
        yield from self._run_half()
        
        # ═══ 比分判定 ═══
        if self.is_tied():
            yield from self._run_overtime()
    
    def _run_half(self):
        for turn in range(self.HALF_TURNS):
            self.turn_in_half = turn
            attacking_team = (self.first_team_this_half + turn) % 2
            defending_team = 1 - attacking_team
            yield from self.execute_turn(attacking_team, defending_team)
    
    def execute_turn(self, atk_team, def_team):
        """一个完整回合——攻守双方都有操作"""
        
        # ═══ Phase 1: 进攻方输入 ═══
        launch_data = yield WaitForPlayerInput(
            team=atk_team,
            prediction_level=self.prediction_level
        )
        
        # ═══ Phase 2: 技能预处理 ═══
        hero = launch_data.hero
        velocity = launch_data.velocity
        
        if hero.active_skill and hero.skill_ready:
            modifier = hero.active_skill.get_modifier()
            velocity = modifier.on_launch(hero, velocity)
            hero.physics_modifier = modifier
        
        # ═══ Phase 3: 弹射 + 防守回应窗口同时开启 ═══
        hero.vel = velocity
        hero.is_active = True
        hero.is_launched = True
        camera.follow(hero, smooth=True)
        
        self.defense_response.team = def_team
        self.defense_response.on_opponent_launch()
        
        # ═══ Phase 4: 物理模拟（攻守并行） ═══
        simulation_time = 0
        shot_attempted = False
        ball_touched = False
        
        while simulation_time < MAX_SIMULATION_TIME:
            physics_engine.update(DT)
            self.defense_response.update(DT)
            
            # 守门员处理（根据模式选择AI或手动）
            for gk in goalkeepers:
                if gk.team == def_team and self.manual_save_enabled:
                    if gk.manual_save.active:
                        pass  # 等待玩家输入
                    elif self._ball_approaching_goal(gk, ball):
                        gk.manual_save.activate(ball)
                else:
                    gk.ai.update(ball, DT)
            
            # 射门检测（用于反消极系统）
            if self._is_shot_attempt(ball, atk_team):
                shot_attempted = True
            if ball.last_touch_hero is not None:
                ball_touched = True
            
            # 进球检测
            if ball.is_in_goal:
                # 手动扑救判定
                gk = self._get_defending_gk(def_team)
                if self.manual_save_enabled and gk.manual_save.active:
                    if gk.manual_save.resolve(ball):
                        ball.is_in_goal = False
                        yield PlaySaveCelebration(gk)
                        continue
                
                yield PlayGoalCelebration(ball)
                break
            
            if physics_engine.all_stopped():
                break
            
            simulation_time += DT
            yield WaitOneFrame()
        
        # ═══ Phase 5: 回合结算 ═══
        for h in self.teams[atk_team].heroes:
            h.update_cooldowns()
        
        max_combo = ball.combo_count
        if max_combo >= 3:
            yield ShowComboBonus(max_combo)
        
        # 反消极系统检查
        self.anti_passive.on_turn_end(atk_team, shot_attempted, ball_touched)
        
        # 逆风buff
        score_diff = abs(self.scores[0] - self.scores[1])
        losing_team = 0 if self.scores[0] < self.scores[1] else 1
        buffs = self.anti_passive.get_comeback_buffs(losing_team, score_diff)
        if buffs:
            self._apply_comeback_buffs(losing_team, buffs)
        
        # 重置
        ball.combo_count = 0
        hero.is_launched = False
        hero.collision_count = 0
```

## 七、物理调参指南（给程序员的备注）

### 7.1 关键参数调整表

| 参数 | 影响 | 调大效果 | 调小效果 | 建议范围 |
|------|------|---------|---------|---------|
| `friction` | 运动持续时间 | 物体滑更远 | 物体很快停下 | 0.95~0.99 |
| `restitution` | 反弹力度 | 弹得更猛 | 反弹很弱 | 0.5~0.95 |
| `MAX_LAUNCH_SPEED` | 弹射力度上限 | 更暴力刺激 | 更精细策略 | 1200~2400 |
| `MAGNUS_COEFFICIENT` | 弧线球弯曲程度 | 球弯得离谱 | 几乎看不出弧线 | 0.08~0.25 |
| `VELOCITY_THRESHOLD` | 回合结束判定 | 快速结束 | 等很久才停 | 3~10 |
| `GK.REACTION_DELAY` | 守门员反应 | 更容易进球 | 守门员太强 | 0.1~0.3 |
| `combo_speed_bonus` | 连击速度加成 | Combo更爽但容易失控 | 连击感不明显 | 5%~15%/次 |
| `intercept_chance` | 拦截率 | 更难射门 | 防守形同虚设 | 20%~50%基础 |

### 7.2 手感黄金比例

```
理想的一次弹射应该持续 2~4 秒
理想的 combo 长度是 2~4 次连击
每场比赛理想时长 3~5 分钟
进球率应该在每 3~4 次弹射进 1 球
```

### 7.3 性能优化要点

```
1. 使用空间哈希网格（Spatial Hash Grid）加速碰撞检测
   - 将场地划分为 100x100 的网格
   - 只检测同一格子和相邻格子内的物体
   
2. 物理更新使用固定时间步长（Fixed Timestep）
   - 渲染帧率和物理帧率解耦
   - 物理固定 60fps，渲染尽可能高
   
3. 轨迹预测线使用简化物理
   - 不计算技能效果
   - 只计算边界反弹
   - 只预测弹射武将轨迹，不预测球
   
4. 碰撞检测使用 Broad Phase + Narrow Phase
   - Broad: AABB 快速排除
   - Narrow: 精确圆-圆检测
```

## 八、物理引擎技术选型建议

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **自研 2D 物理** | 完全可控，轻量 | 需要自己处理各种边界情况 | ⭐⭐⭐⭐ 推荐 |
| Unity 2D Physics (Box2D) | 成熟稳定，编辑器友好 | 对弹射游戏过于"真实"，需要大量调参 | ⭐⭐⭐ |
| Godot Physics 2D | 开源免费，够用 | 社区相对小 | ⭐⭐⭐ |
| SpriteKit (iOS原生) | 苹果原生性能最优 | 只能iOS，不跨平台 | ⭐⭐ |

> **推荐：** 使用 Unity/Godot 作为游戏引擎，但物理部分 **自己写**（如本文档所述），不用引擎自带物理。
> 原因：弹射类游戏需要的不是"真实物理"而是"好玩的物理"，自研更容易调出好手感。
