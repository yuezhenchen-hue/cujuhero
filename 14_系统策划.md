| Lv.25 | 赛事系统 | 竞技深度系统 |
| PVP鞠圣(钻石) | 手动扑救、完整BP | 技术能力验证 |

## 二、招贤纳士系统（抽卡）

### 2.1 系统流程

```
         ┌──────────┐
         │ 招贤大厅  │
         └────┬─────┘
              │
    ┌─────────┼─────────┐
    ▼         ▼         ▼
┌────────┐ ┌────────┐ ┌────────┐
│常驻池  │ │UP池    │ │限定池  │
│(永久)  │ │(2周轮换)│ │(活动)  │
└────┬───┘ └────┬───┘ └────┬───┘
     │          │          │
     ▼          ▼          ▼
  ┌──────────────────────────┐
  │      抽卡动画演出         │
  │  N=绿光  R=蓝光          │
  │  SR=紫光  SSR=金光+全屏   │
  └────────────┬─────────────┘
               │
     ┌─────────┼─────────┐
     ▼         ▼         ▼
  新角色     重复角色     碎片
  (首次获得)  (转换碎片)   (直接获得)
```

### 2.2 卡池类型

| 卡池 | 常驻时间 | 特点 | 保底共享 |
|------|---------|------|---------|
| **梁山聚义池**（常驻） | 永久 | 包含所有非限定角色 | 独立计数 |
| **英雄UP池** | 每2周轮换 | 指定1名SSR+2名SR概率提升 | 独立计数 |
| **限定池** | 活动期间 | 限定角色（如世界杯限定、节日限定） | 独立计数 |
| **新手池** | 仅首周 | 前20抽必出SSR，仅限一次 | 独立计数 |

### 2.3 抽卡交互设计

```
单抽操作流程：
  1. 点击「单抽」按钮（消耗160银两或1酒令）
  2. 画面切换到蹴鞠场景——一个蹴鞠从天而降
  3. 蹴鞠颜色暗示品质：
     绿色旋转 = N
     蓝色旋转 = R
     紫色旋转 + 闪电 = SR
     金色旋转 + 全屏金光 + 鼓声 = SSR
  4. 蹴鞠落地炸开，角色从中走出
  5. 展示角色卡面 + 属性概要
  6. 点击确认收入

十连操作流程：
  1. 点击「十连」（消耗1440银两或10酒令）
  2. 10个蹴鞠从天而降，排成一排
  3. 按品质从低到高依次开启（N→R→SR→SSR）
  4. SSR出现时全屏特写演出
  5. 最终展示10张卡结果一览
  6. 「跳过动画」按钮（连续抽卡时可用）

重复角色转化：
  重复N → 该角色碎片 ×5
  重复R → 该角色碎片 ×10
  重复SR → 该角色碎片 ×15 + 通用碎片 ×5
  重复SSR → 该角色碎片 ×30 + 通用碎片 ×15 + 星宿令 ×5
```

### 2.4 保底UI显示

```
抽卡界面右上角常驻显示：
  ┌───────────────────────┐
  │ 距SSR保底：47/80抽     │  ← 明确告知玩家距离保底的距离
  │ 距SR保底：3/10抽       │
  │ 星宿令：128/200        │  ← 天井进度
  │ UP保底状态：未触发      │  ← 告知是否"歪过"
  └───────────────────────┘

设计原则：
  保底计数完全透明，不做任何隐藏
  → 减少玩家焦虑，增加消费决策的确定性
  → 合规要求（中国版号概率公示）
```

## 三、装备系统

### 3.1 系统结构

```
装备获取
  │
  ├── PVE关卡掉落
  ├── Roguelike宝箱
  ├── PVP赛季奖励
  ├── 商店购买
  └── 合成锻造
       │
       ▼
┌─────────────────────────────────┐
│           装备背包               │
│  容量上限：200件                 │
│  排序：品质/属性/槽位/获取时间   │
│  筛选：按槽位/品质/主属性        │
│  批量分解：一键分解白绿蓝        │
└──────────┬──────────────────────┘
           │
     ┌─────┼─────┐
     ▼     ▼     ▼
  穿戴   强化   分解
```

### 3.2 装备穿戴规则

```
每个武将4个装备槽：
  [蹴鞠靴] — 脚部，主加STR/SPD
  [护  腕] — 手部，主加DEF/WIL
  [球  衣] — 身体，主加全属性
  [腰  带] — 腰部，主加TEC/WIL

穿戴限制：
  · 装备无角色绑定，可自由摘下换给他人
  · 无等级限制（任何装备任何角色都能穿）
  · 无位置限制（前锋也能穿防守装备）
  · 一件装备同时只能被一个角色穿戴

装备套装效果（仅红色品质）：
  「禁军铠甲」套装(4件)：全属性+10，被碰撞击退-30%
  「梁山义服」套装(4件)：全属性+8，羁绊效果+20%
  「水墨华服」套装(4件)：TEC+15，弧线球效果+25%
  → 套装件数不足时无额外效果
```

### 3.3 装备强化流程

```
强化消耗：铜钱 + 强化石
强化等级：+0 → +15

强化消耗表：
  +1~+3：铜钱500/次，强化石×1，100%成功
  +4~+6：铜钱1500/次，强化石×2，100%成功
  +7~+9：铜钱3000/次，强化石×3，80%成功（失败不降级）
  +10~+12：铜钱6000/次，强化石×5，60%成功（失败不降级）
  +13~+15：铜钱12000/次，强化石×8，40%成功（失败降1级）

每+3强化，随机强化一条副词条+20%
  +3：第1条副词条强化
  +6：第2条副词条强化
  +9：第3条副词条强化
  +12：第4条副词条强化（紫品以上）
  +15：随机一条副词条额外+20%（满级特效）

装备分解产出：
  白色 → 强化石×1 + 铜钱100
  绿色 → 强化石×3 + 铜钱300
  蓝色 → 强化石×6 + 铜钱800
  紫色 → 强化石×12 + 铜钱2000
  金色 → 强化石×25 + 铜钱5000
  红色 → 强化石×50 + 铜钱12000 + 套装碎片×1
```

### 3.4 装备合成锻造

```
合成公式：
  3件同品质装备 → 1件高一级品质装备（随机属性）
  例：3件蓝色蹴鞠靴 → 1件紫色蹴鞠靴

指定合成（消耗额外铜钱）：
  锁定主属性：额外消耗铜钱×3
  锁定槽位：额外消耗铜钱×2
  
  例：3件蓝色 + 15000铜钱 → 1件紫色蹴鞠靴(指定STR主属性)
```

## 四、蹴鞠会馆（公会系统）

### 4.1 系统结构

```

│  └─────────┘  └─────────┘  └─────────┘  │
└───────────────────────────────────────────┘
```

### 4.2 会馆创建与管理

```
创建条件：Lv.20 + 50,000铜钱
加入条件：Lv.15
会馆容量：初始30人，升级可扩至50人
管理层级：
  馆主（1人）— 全部权限
  副馆主（2人）— 除转让和解散外全部权限
  教头（5人）— 审批入馆、踢人、修改公告
  馆员（无限）— 基础参与权限

会馆等级与特权：
  Lv.1（初始）：30人容量，基础聊天
  Lv.2（贡献5000）：35人容量，馆标自定义
  Lv.3（贡献15000）：40人容量，助战系统解锁
  Lv.4（贡献30000）：45人容量，会馆联赛解锁
  Lv.5（贡献60000）：50人容量，集体任务解锁，会馆商店解锁
```

### 4.3 会馆联赛

```
赛制：每周一轮联赛
  周一~周五：报名阶段，馆主选出5名代表
  周六：联赛日
    → 系统随机匹配另一个会馆
    → 5v5车轮战（每人打一场PVP，3胜制）
    → 天梯赛标准化规则
  
联赛积分：
  胜利：+3分
  平局：+1分
  失败：+0分

赛季排名奖励（每60天结算）：
  前10%：全馆成员获得限定称号 + SSR碎片×5
  前30%：全馆成员获得SR碎片×10 + 银两×200
  前50%：全馆成员获得铜钱×50,000
```

### 4.4 助战系统

```
功能：借用馆友的武将
  · 每天可借用1次馆友武将用于PVE/Roguelike
  · 借用武将使用馆友的养成数据（等级/星级/装备）
  · 被借用方获得铜钱奖励（越强的武将奖励越高）
  · PVP不可借用

借出奖励：
  每次被借用 → 馆主获得铜钱1000 + 会馆贡献10
  每日被借用上限：10次
```

### 4.5 集体任务

```
每周集体任务（全馆协作）：
  · "全馆累计进球100个" → 铜钱×30,000（全馆分配）
  · "全馆累计PVP胜利50场" → 银两×50（全馆分配）
  · "全馆累计combo 500次" → 经验丹×50（全馆分配）
  
每日会馆签到：
  · 每人每天签到获得铜钱1000 + 会馆贡献5
  · 全馆当日签到人数≥20 → 额外全馆银两×10
```

### 12.1 存档策略

```

  · 操作设置、画面设置
  · 离线比赛记录缓存
  · 教学进度标记

云端存档（服务器）：
  · 玩家所有角色数据
  · 装备数据
  · 货币余额
  · 抽卡保底计数
  · PVP段位
  · 成就进度
  · 任务进度
  · 邮件
  · 好友列表

存档冲突处理：
  · 以服务器数据为准
  · 客户端不可直接修改关键数值（防作弊）
  · 离线操作产生的变更在联网后上传校验
```

### 12.2 防作弊设计

```
客户端-服务器校验点：
  1. 抽卡结果：服务器生成随机数，客户端只接收结果
  2. PVP战斗：帧同步，双方输入在服务器仲裁
  3. 货币变动：所有增减都经服务器验证
  4. PVE通关：客户端上报操作序列，服务器可重放验证
  5. 装备强化：随机结果由服务器生成

客户端防篡改：
  · 内存加密（关键数值异或加密）
  · 完整性校验（定期比对本地/服务器数据）
  · 越狱/Root检测（降级到离线模式，禁止PVP）
```
