## 一、技术选型

### 1.1 游戏引擎选型

| 方案 | 优点 | 缺点 | 适合度 |
|------|------|------|--------|
| **Unity 2D** | 生态成熟，iOS/Android双端，Asset Store丰富，Spine支持好 | 包体偏大，运行时GC | ⭐⭐⭐⭐⭐ **首选** |
| Godot 4 | 开源免费无分成，GDScript易学，2D性能优秀 | 生态不如Unity，Spine插件需额外配 | ⭐⭐⭐⭐ |
| Cocos Creator | 国内生态好，包体小，H5/小程序/iOS/Android全覆盖 | 3D支持弱，社区缩小中 | ⭐⭐⭐ |
| SpriteKit (原生) | 苹果原生性能极致，与iOS无缝集成 | 只支持iOS，开发效率低 | ⭐⭐ |
| Unreal Engine | 画面顶级 | 2D游戏用UE是大炮打蚊子 | ⭐ |

**推荐方案：Unity 2022 LTS + C#**

理由：
1. iOS和Android双端发布，一套代码
2. 2D Sprite/Spine骨骼动画支持成熟
3. 物理部分自研（不用Unity Physics 2D），引擎只负责渲染和资源管理
4. 丰富的插件生态：Spine Runtime, DOTween, UniTask等
5. App Store / Google Play 发布流程成熟

```
│                    表现层                        │

├────────────────────────────────────────────────┤
│                    逻辑层                        │

│                    物理层                        │

│  碰撞检测 → 碰撞响应 → 弹射系统                │
│  轨迹预测 → 马格努斯效应 → 守门员AI             │
├────────────────────────────────────────────────┤
│                    数据层                        │
│  角色数据(ScriptableObject) → 关卡配置(JSON)    │

└────────────────────────────────────────────────┘
```

### 1.3 后端技术栈

```
服务器框架：  Go (Gin) 或 Python (FastAPI)
数据库：      PostgreSQL（主库）+ Redis（缓存/排行榜）
实时通信：    WebSocket (PVP对战)
对象存储：    阿里云OSS / AWS S3（资源CDN）
运维：        Docker + Kubernetes
日志/监控：   Grafana + Prometheus
推送：        APNs (iOS) + FCM (Android)
```

| 用途 | 推荐方案 | 说明 |
|------|---------|------|

| 异步框架 | UniTask | 替代协程 |

## 四、关键技术难点与解决方案

### 4.1 弹射手感调优

```
难点：弹射手感是核心体验，做不好就没人玩
方案：
  1. 物理引擎自研，不依赖Unity Physics
  2. 建立参数调试面板（编辑器内可实时调参）
  3. 录制弹射replay，用于分析和回放
  4. 建立A/B测试框架，让不同玩家体验不同参数
  
关键参数调试工具：
  - 力度曲线编辑器（AnimationCurve）
  - 摩擦/弹性实时滑块
  - 慢动作调试模式（0.1x~2.0x速度）
  - 碰撞可视化（显示碰撞体、力向量）
```

### 4.2 PVP网络同步

```
难点：回合制弹射需要精确同步物理结果
方案：
  采用"帧同步"方案（非状态同步）
  
  1. 双方客户端运行相同的确定性物理引擎
  2. 只同步操作输入（弹射方向+力度）
  3. 使用定点数(Fixed Point)替代浮点数，保证跨平台一致
  4. 每回合结束时校验双方状态哈希，不一致则以服务器仲裁
  
  因为是回合制，网络延迟容忍度高（500ms以内都可接受）
  大幅降低了实时同步的技术难度
```

### 4.3 性能优化

```
目标设备：iPhone 12 (A14) 及以上
目标帧率：60fps稳定

1. 物理引擎优化：
   - 空间哈希网格碰撞检测（O(n) 而非 O(n²)）
   - 静止物体跳过物理更新
   - 碰撞检测使用 Broad Phase (AABB) + Narrow Phase (Circle)
   
2. 渲染优化：
   - Sprite Atlas 合图减少 Draw Call
   - 动态合批（Dynamic Batching）
   - 特效使用GPU粒子
   - 远处物体LOD降级
   
3. 内存优化：
   - Addressables按需加载/卸载资源
   - Spine动画使用共享Atlas
   - 对象池（Object Pool）复用子弹/特效/粒子
   
4. 包体优化：
   - 首包只含第一章资源，其余按需下载
   - 音频使用Opus格式压缩
   - 贴图使用ASTC压缩（iOS原生支持）
```
