## 二、障碍物系统

### 2.1 障碍物类型

| 类型 | 形状 | 碰撞属性 | 出现时机 | 视觉 |
|------|------|---------|---------|------|
| **石柱** | 圆形 r=25 | 固定，弹性0.8 | 第二章起 | 青石柱 |
| **木箱** | 方形 60×60 | 固定，弹性0.6 | 第一章起 | 货物箱 |
| **水坑** | 圆形 r=50 | 非碰撞，减速区域 | 第二章起 | 水花涟漪 |
| **沙堆** | 圆形 r=40 | 非碰撞，高摩擦区域 | 第三章起 | 黄沙堆 |
| **活动墙** | 方形 120×30 | 固定，每2回合移动 | 第三章起 | 木板屏风 |
| **弹射板** | 方形 80×20 | 固定，弹性1.5（超弹） | 第三章起 | 弹簧板 |
| **陷阱** | 圆形 r=35 | 非碰撞，眩晕1回合 | 第二章起 | 地刺陷阱 |
| **BOSS专属** | 各异 | 各异 | BOSS关 | 见BOSS设计 |

### 2.2 障碍物放置原则

```
设计原则：
  1. 障碍物必须「可利用」——不仅是阻碍，也是策略工具
     · 石柱可用于反弹绕过防守
     · 弹射板可用于加速射门
     · 水坑可诱骗敌方踩入减速
  
  2. 障碍物不能阻断「必经之路」
     · 球门前方不放不可破坏的障碍物
     · 至少保留2条射门路线
  
  3. 逐章递增
     · 第一章：0~1个障碍物（纯蹴鞠体验）
     · 第二章：1~2个（引入障碍物概念）
     · 第三章：2~3个（策略复杂度提升）
     · 第四章：3~5个（BOSS战最复杂）
  
  4. 对称与非对称
     · 普通关卡：障碍物左右对称（公平）
     · BOSS关卡：障碍物非对称（增加BOSS方优势）
```

### 3.3 第二章重点关卡

#### 2-4「不公正的裁判」

```
场景：济州府蹴鞠场
特殊规则：裁判偏袒

机制说明：
  · 我方每次进球，30%概率被"判无效"（裁判吹哨取消进球）
  · 对方进球100%有效
  · 需要用"无可争议的进球"才能确保得分：
    → combo ≥ 3的进球不会被取消
    → 弧线球进球不会被取消
    → 观众欢呼值达到80%以上时不会被取消

观众欢呼值系统（本关特殊）：
  · 初始50%
  · 每次华丽操作（combo、弧线球、反弹进球）+10%
  · 每次普通操作 +2%
  · 被进球 -15%
  · 欢呼值≥80%时裁判被"民意"压制，不再判无效

胜利条件：比分胜利（2分领先）
回合数：12

设计意图：
  教会玩家"combo不仅是得分倍率，也是战术保障"
  与叙事配合——蔡京的阴谋需要用实力破解
```

### 4.1 地图生成算法

```
地图结构：有向无环图（DAG）
节点生成规则：

def generate_roguelike_map(floor_level):
    """生成一层Roguelike地图"""
    
    config = FLOOR_CONFIG[floor_level]
    # floor_level 1: nodes=4~5, rows=4
    # floor_level 2: nodes=5~6, rows=5
    # floor_level 3: nodes=6~7, rows=6
    # floor_level 4: nodes=3~4, rows=3
    
    rows = config['rows']
    nodes_per_row = [random.randint(1, 3) for _ in range(rows)]
    nodes_per_row[0] = 1    # 起点行只有1个节点
    nodes_per_row[-1] = 1   # BOSS行只有1个节点
    
    # 生成节点
    for row_idx, count in enumerate(nodes_per_row):
        for node_idx in range(count):
            node_type = pick_node_type(row_idx, rows, floor_level)
            # ...
    
    # 生成边（保证从起点到BOSS至少有2条不同路径）
    for row_idx in range(rows - 1):
        for node in nodes[row_idx]:
            targets = pick_connections(nodes[row_idx + 1], min=1, max=2)
            # ...
    
    return map_graph


def pick_node_type(row_idx, total_rows, floor_level):
    """根据行位置和层数选择节点类型"""
    
    if row_idx == 0:
        return 'start'
    if row_idx == total_rows - 1:
        return 'boss'
    
    # 节点类型权重
    weights = {
        'battle':  40,  # 比赛
        'event':   20,  # 随机事件
        'rest':    15,  # 休息站
        'shop':    15,  # 商店
        'treasure': 10, # 宝箱
    }
    
    # 层数越高，比赛和事件越多，休息越少
    weights['battle'] += floor_level * 5
    weights['rest'] -= floor_level * 3
    
    # 保证每层至少1个休息站和1个商店
    # ...
    
    return weighted_random(weights)
```
